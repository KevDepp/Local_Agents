# Task T-001_hello — Créer `hello.txt`

## Contexte

L'utilisateur souhaite une séquence simple de création de fichiers :
1) Créer `hello.txt`.
2) Créer `world.txt`.
3) Créer `files.md` listant les deux fichiers.

Cette tâche couvre uniquement l'étape 1 : créer `hello.txt`.

## Objectif

Créer le fichier `hello.txt` à la racine du projet (`cwd`) comme premier élément du pipeline de fichiers.

## Exigences

- Créer un fichier nommé `hello.txt` à la racine du projet.
- Ne pas créer `world.txt` ni `files.md` dans cette tâche.
- Garder les changements minimaux et cohérents avec les instructions du projet.

## Definition of Done

- `hello.txt` existe à la racine du projet.
- L'item de checklist correspondant à T-001_hello dans `doc/TESTING_PLAN.md` est satisfait.
- `data/tasks/T-001_hello/dev_result.md` (ou `.json`) décrit clairement les commandes utilisées pour créer/vérifier le fichier (par ex. `Test-Path hello.txt`) et leurs sorties.
- Aucun fichier sans lien avec cette tâche n'est modifié.
